Nodes

🔍 classify_node  sync

Classifies user_input using a deterministic LLM (temperature=0). Returns one of three labels. Errors default to "on_topic" to avoid false positives on the off-topic guard.

LabelMeaningExample
ON_TOPICOS-related question"What is paging?"
CASUALSocial / emotional message"Good morning!", "Thanks!"
OFF_TOPICUnrelated to OS"Solve 2+2", "Write a poem"

📌 off_topic_node  no LLM call

Generates a fixed template response — no LLM involved. Uses the first 6 words of user_input as the topic label. Routes directly to END, skipping parse_visual_node.

"📌 Just so you know, **{topic}** is not part of this Operating Systems course.\n\nBut if you're curious, I'm happy to explain it as a special request! 😊"

🎓 instructor_node  async

Streams the teaching response. Two internal sub-paths:

Sub-pathConditionBehaviour
A — Off-topic explainoriginal_off_topic is setBrief friendly explanation of the off-topic subject, then "Now, back to OS! 📘"
B — Normal teachingDefaultPicks _MODE_SYSTEM_PROMPTS[mode] and passes full conversation history. casual messages also take this path and get a warm 1–3 sentence reply.

🖼 parse_visual_node  sync

Post-processes the instructor's response. Searches for [VISUAL:tag_name:args] using regex \[VISUAL:([a-z_]+)(?::([^\]]*))?\]. If found, strips the tag from response and populates visual_tag + visual_args. These are serialised into a VisualPayload and sent in the SSE done event.

SSE Streaming Protocol

chat.py calls chat_graph.astream_events(input_state, version="v2") and maps LangGraph events to SSE frames sent to the React frontend.

LangGraph EventSSE Frame
on_chat_model_stream from instructor_node {"type":"token","delta":"..."}
on_chain_end + classification == "off_topic" {"type":"offtopic","content":"..."}
on_chain_end + normal finish {"type":"done","content":"...","visual":{...}}
Any exception {"type":"error","message":"..."}

Tokens are streamed only from instructor_node — classify_node tokens are suppressed via langgraph_node metadata filter.
